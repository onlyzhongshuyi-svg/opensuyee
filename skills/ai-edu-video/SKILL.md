# AI 科普视频

用这个 skill 制作适合传播的 AI 科普/教程类短视频。核心目标是：画面内容和口播逐句对应，信息清晰，小白能看懂，成片适合视频号、小红书等平台发布。

这个 skill 的核心规则写在 `SKILL.md`，同时提供跨 Agent 入口文件：`AGENTS.md`、`CLAUDE.md` 和 `references/cross-agent-usage.md`。在 Codex、Claude Code、opencode、OpenClaw 或其他 Agent 中使用时，都应先读取本文件，再读取对应模板文件。

优先使用可复用的程序化视频流程，例如 Python/Pillow/ffmpeg、Remotion 或 HyperFrames。这样可以稳定控制版式、字幕、时长、动效、抽帧检查和多轮修改。

## 全局规格

- 默认视频比例统一为 **3:4 竖屏**，推荐导出 `1080x1440`。
- 只有当用户明确指定其他比例、平台要求不同，或源素材必须保留特殊比例时，才改用其他画幅。
- 所有模板都要先按 3:4 安全区设计标题、字幕、口播 PiP、进度条和关键视觉元素。

## 工作流

1. 收集输入：
   - 口播稿、逐字稿或已录好的口播视频。
   - 主讲人口播视频或音频。
   - 产品名称、产品 icon/logo、品牌色、截图、录屏、官网素材或参考风格。
   - 发布平台、视频比例、是否要先出设计方案。
2. 如果用户要求“先给方案”，先输出具体视频设计方案，等用户确认后再制作。
3. 拆分口播稿并建立精确时间轴：
   - 先从口播视频/音频提取音频，转写成逐字稿；如果用户已经给出口播稿，要和音频核对，修正漏词、口头语和顺序差异。
   - 按语义短句切分口播稿：逗号、句号、转折词、步骤词、结果承诺、产品名出现处都可以作为切点。
   - 把短句合并成场景：每个场景只讲一个核心意思；每个场景内可以有多个逐步出现的模块。
   - 为每个短句或模块建立时间轴字段：`start`、`end`、`spoken_text`、`caption_text`、`visual_action`、`scene_id`。
   - 优先使用真实语音时间戳；如果没有自动识别时间戳，先按音频总时长和每句字数估算，再用波形、试听、抽帧逐段校准。
   - 画面模块必须对应当前正在说的句子，不能提前展示后面才讲的内容。
   - 字幕时间轴要比画面更细，按当前口播短句显示，不能把整段口播堆在同一帧。
4. 从 `references/` 选择一个模板。
5. 制作视频：
   - 生成静态或动态场景帧。
   - 按需加入右下角口播 PiP。
   - 按严格字数规则生成字幕。
   - 有章节时加入顶部进度条或章节提示。
   - 使用原口播音频合成最终视频。
6. 交付前验收：
   - 检查宽高、帧率、音频流、总时长、文件大小。
   - 抽取代表帧：封面、封面后正文、章节切换点、用户指定的时间点。
   - 检查文字是否溢出、素材是否缺失、logo/icon 是否被遮挡、水印是否拥挤、字幕是否堆叠、画面和口播是否错位。

## 模板列表

- **apple-green-pip-template**：读取 `references/apple-green-pip-template.md`。这是这次沉淀的第一种模板：干净、清晰、偏 Apple 官网教程感，主色绿色，封面大标题，顶部章节进度条，右下角口播 PiP，字幕黑底自适应长度，模块逐步出现。适合 AI 工具入门教程、产品科普、零基础操作教学。
- **dark-kinetic-course-template**：读取 `references/dark-kinetic-course-template.md`。深色高能课程感模板：真人口播顶部 PiP、暗色科技背景、大字号动效标题、黄色描边字幕、章节角标、难度条/评分/截图揭示等强节奏动效。适合更强调留存、冲击力和口播重点同步的 AI 工具教程、知识付费风格课程切片、创作者 explainer。

## 产品 Icon 规则

- 不要写死某一个产品的 icon。
- 根据本期视频讲的产品选择 icon/logo。
- 如果用户提供了产品 icon/logo，优先使用用户提供的素材。
- 如果用户没有提供 icon/logo：
  - 可以先问用户是否需要补充素材；
  - 或使用产品名称的文字标识、简洁几何符号、截图中的产品标识作为替代；
  - 不要擅自使用不确定来源或不匹配的品牌图标。
- icon/logo 只能作为封面或关键解释页的辅助识别元素，不能挤压标题、字幕或口播 PiP。

## 字幕规则

- 除非用户明确修改规则，每一行中文字幕不超过 20 个字。
- 每帧字幕尽量 1 行，必要时最多 2 行。
- 字幕黑色底框宽度跟随字幕实际长度，并限制在安全区域内。
- 当画面右下角有口播 PiP 时，字幕优先放在左侧空白安全区域，并在该区域内水平居中，而不是贴左边缘。
- 字幕字号要适合小红书/视频号竖屏观看，默认比正文说明文字更醒目；放大字号后必须检查不压到口播 PiP、进度条或主标题。
- 字幕文本在黑色底框内也要居中对齐，避免短字幕看起来偏左。
- 不要把大段口播稿堆成字幕块，字幕只承接当前正在说的话。

## 拆稿与精确时间轴规则

- 先做“句子级时间轴”，再做“场景级画面设计”。
- 句子级时间轴至少包含：
  - `start`：这句话开始的秒数。
  - `end`：这句话结束的秒数。
  - `spoken_text`：口播原句。
  - `caption_text`：字幕文本，可比口播更精炼，但不能改意思。
  - `visual_action`：这一句话对应的画面动作，例如出现标题、揭示卡片、切换截图、推进进度条。
  - `scene_id`：所属场景。
- 场景级时间轴由多个句子组成，每个场景要有 `scene_start`、`scene_end`、`title`、`visual_type`、`stage_count`。
- 校准时间轴时按这个顺序做：
  - 用语音识别或人工听写得到初稿时间戳。
  - 对照用户提供的正式口播稿修正文本。
  - 根据语气停顿和视觉节奏合并过短句子。
  - 导出后抽取关键时间点帧，检查画面是否对应当前口播。
- 用户指出错位时间点时，以用户时间点为最高优先级重新调整。
- 如果自动语音识别不可用，也必须建立估算时间轴，并在最终回复中说明时间轴是估算后抽帧校准的。

## 用户反馈后的时间轴修正方法

- 用户指出具体时间点时，把该时间点作为最高优先级校准锚点。先把 `1:08` 这类时间转成秒和帧，再检查 `t - 0.5s`、`t`、`t + 0.5s` 的画面。
- 先判断被指出的问题属于哪一层：主标题、章节角标、正文模块、字幕、口播 PiP 或进度条。默认主标题可以提前暴露关键词；正文模块、步骤卡片、flow、terminal、badge、数字动画必须等口播说到对应词再出现。用户明确指出标题或章节角标时，以用户要求为准。
- 不要用“重点”等新文案掩盖错位。优先保留用户认可的标题/模块文案，通过拆分时间轴、调整 `visibleItems` 或绝对时间控制来解决提前出现。
- 如果关键词出现在一句话后半段，把这句话拆成更细的字幕/slide。例如先显示“给大家讲”，等口播到“Codex 是什么”再显示 `01 Codex 是什么` 模块。
- 对 `01/02/03` 这类步骤模块，出现后默认保持可见，后续步骤叠加出现，除非用户明确要求替换或隐藏。
- 不要因为过滤未说到的副文案而删除整张卡片再重新编号；这会导致 `02` 变成 `01`。应保留卡片顺序和编号，只隐藏未说到的副文案，或把卡片整体 reveal 时间后移。
- 当一个视觉段跨越多个 scene/slide 时，用绝对时间 `t` 驱动标题、章节角标、模块和数字动画，避免 scene-local `localFrame` 在切换时重置造成元素闪烁或消失。
- 数字/进度类强调适合做成独立动画，例如空圆环在 1 秒内填到 `90%`。动画前、动画中、动画结束后的画面都要抽帧检查。
- flow/对比卡片需要表达“左侧对象 -> 右侧角色/结果”时，右侧短词（如 `agent`、`chat`、`自然语言`）放在横框右半部分，不要默认堆在标题下方。
- 修改完成后先用 still 抽用户指出的时间点和前后帧，确认通过后再全量渲染、合成口播 PiP，并从最终 MP4 再抽同样时间点复查。

## 时间与动效规则

- 同一个页面的不同模块不要一次性全出现，要跟随口播逐步出现。
- 重点词视觉强调必须和口播同步：如果要做放大、缩放回位、高亮框、弹出卡片等 spotlight 动效，必须绑定到包含该词的字幕/口播短句时间轴，不能在用户说出该词之前展示。
- 避免“视觉提前剧透”：在重点词正式出现前，标题、模块、卡片、流程图、终端示例、标签等画面元素也不要提前露出真实关键词。必要时先用中性占位文案，例如“重点”“关键步骤”“第一个项目”“最难部分已搞定”，等口播说到该词时再替换成真实词并做强调。
- 对所有做了 spotlight 的词，导出后至少抽取“出现前 0.5-1 秒”和“出现时”的成片帧做对照检查：出现前不能看到真实重点词或放大态，出现时字幕和重点词必须一致。
- 用户要求封面式开场时，前 1-3 秒可以使用独立封面排版，不必沿用正文布局。
- 用户指出某个时间点有问题时，要保留这个时间点修复要求，并在导出后抽帧检查。
