# Dark Kinetic Course 模板

这个模板来自一个深色、高节奏、强口播同步的 AI 教程参考视频。它不走极简官网感，而是更接近短视频平台的“高留存课程切片”：真人持续在场，文字冲击更强，重点词有明显动效，适合视频号、小红书、抖音等信息密度高的教程内容。

## 适用场景

- AI 工具入门教程、效率工具教程、产品科普、知识付费课程切片。
- 用户希望视频更有冲击力、重点更容易被看见，而不是纯净克制的品牌官网风。
- 口播内容有明确章节、观点、痛点、数字、评分、步骤、实操截图时优先考虑。

## 视觉规格

- 视频比例：默认 `3:4` 竖屏，推荐导出 `1080x1440`。
- 背景：深黑、深蓝、深青为主，使用径向暗角、轻微噪点、局部青绿色/蓝色光晕。
- 主色建议：
  - 背景：`#050B10`、`#07141A`
  - 高亮青绿：`#26E6B1`
  - 高亮橙：`#FFAA22`
  - 高亮蓝：`#3F9BFF`
  - 字幕黄：`#FFD21A`
  - 正文白：`#F8FAFC`
  - 辅助灰：`#A7B0BC`
  - 警示红：`#E5212A`
- 字体：中文标题使用粗黑体或等价粗体，字重足够重；英文标签用小号大写，作为课程感装饰。
- 不复用参考视频的人物、原文案或具体品牌元素，只沉淀版式、节奏和动效方法。

## 画面结构

- **开头 Hook**：
  - 前 1-3 秒可以用真人口播全屏或接近全屏。
  - 底部直接上大号黄色描边字幕，快速建立真人可信度。
- **正文 PiP**：
  - 讲师视频通常放在顶部居中，宽度约画面宽度的 45%-55%。
  - 使用圆角矩形裁切、轻微阴影或细微辉光。
  - PiP 要稳定，不要频繁换角落；除非进入截图实操页需要避让。
- **章节角标**：
  - 右上角放小型章节 chip，包含编号、英文小标题和中文章节名。
  - 示例结构：`01 WHAT IT IS / 产品是什么`、`02 SET IT UP / 如何安装`。
  - chip 可用青绿、蓝色或橙色细描边和轻微发光。
- **主内容区**：
  - 中部使用大字号标题，按 1-3 行断句。
  - 关键词用橙色、青绿或蓝色加粗高亮。
  - 文字量要少，宁可拆成多个阶段逐步出现，也不要一页塞满。
- **字幕区**：
  - 底部字幕使用黄色粗体 + 黑色描边/阴影。
  - 这个模板默认不使用黑色底框，除非用户明确要求。
  - 每行仍不超过 20 个字，优先 1 行，必要时最多 2 行。
  - 字幕要避开平台底部 UI 安全区，不要贴底裁切。

## 典型场景

- **真人 Hook**：真人全屏 + 大字幕，只承接一句最强开场。
- **痛点页**：英文 eyebrow，例如 `THE PROBLEM`；大标题表达痛点；关键词橙色高亮；周围加入倾斜代码符号、标签、叉号或警示元素。
- **章节页**：中间出现 `PART · 01` 胶囊标签；产品名或章节名巨大显示；下方逐行出现 2-3 个问题。
- **概念定义页**：产品名大字 + 关键词解释；可用对比词、箭头、agent/assistant 等标签辅助理解。
- **能力证明页**：奖杯、星标、排名、上限、评分等符号做中心视觉锚点。
- **难度/评分页**：大号数字评分，例如 `9 /10`；下方难度条从左到右扫过；口播说到数字时数字再弹出。
- **实操截图页**：上方保留暗色标题栏和 PiP，下方展示网页、软件界面或录屏截图；截图用遮罩向上滑入或逐步放大到可读状态。
- **总结 CTA 页**：大标题总结用户已经获得的结果，底部用一条短 CTA 承接关注或下一期内容。

## 动效规则

- 标题不要整页一起出现。按“标签 -> 第一行 -> 高亮词 -> 第二行 -> 辅助元素”的顺序逐步出现。
- 重点词必须绑定真实口播时间点，不能提前出现。口播说到该词时再做放大、弹出、变色、描边或落位。
- 适合 spotlight 的内容：
  - 数字承诺：`30分钟`、`第一个项目`、`9/10`
  - 低门槛词：`零代码`、`零门槛`、`零废话`
  - 关键概念：`agent`、`安装`、`权限`、`项目`
  - 结果词：`官网`、`PPT`、`APP`、`上线赚钱`
- spotlight 推荐动作：
  - 初始 120%-160% 放大入场，占画面中部 20%-35%。
  - 0.25-0.45 秒内回落到最终位置。
  - 使用轻微 overshoot、发光或描边闪烁，避免只做淡入。
- 辅助元素可做轻微漂浮、倾斜、旋转或视差，但不要抢主标题。
- 难度条、进度条、评分条用横向扫光或填充动画；数字用计数或弹跳入场。
- 截图和录屏用遮罩揭示、缩放推进、局部高亮框，不要硬切成静态图。
- 转场节奏短，通常 0.25-0.45 秒；可用推入、快速缩放、轻微模糊，但不要影响文字可读性。

## 口播同步工作流

- 先建立句子级时间轴，再决定每个 spotlight 出现在哪一帧。
- 每个场景至少标记：
  - `scene_start`
  - `scene_end`
  - `section_label`
  - `headline`
  - `spotlight_terms`
  - `pip_layout`
  - `caption_style`
- 每个句子至少标记：
  - `start`
  - `end`
  - `spoken_text`
  - `caption_text`
  - `visual_action`
  - `spotlight_term`
- 如果口播还没说到重点词，画面只能显示中性占位或上一阶段内容。
- 导出后对每个 spotlight 至少抽取两帧：
  - 出现前 0.5-1 秒：不能看到真实重点词。
  - 出现时：重点词、字幕、口播语义必须一致。

## 制作建议

- 用 Remotion、HyperFrames 或 HTML + GSAP 最容易做出这种动效；如果只用 Python/Pillow，也要用分阶段关键帧模拟弹出、缩放、扫光和逐字出现。
- 把页面拆成独立图层：背景、PiP、章节 chip、主标题、关键词、高亮装饰、字幕、截图/录屏。
- 录屏截图要优先保证可读性。必要时裁切关键区域，而不是整页缩得太小。
- 字幕在这个模板里是强视觉元素，不要做得过小；同时保持每行 20 字以内。

## 验收清单

- `ffprobe` 检查成片为 `1080x1440` 或其他 3:4 竖屏尺寸，并包含音频流。
- 抽帧检查：
  - 1 秒：真人 Hook、字幕是否清楚。
  - 每个章节切换点：章节 chip 是否正确。
  - 每个 spotlight 前 0.5-1 秒和出现时：是否严格跟口播同步。
  - 每个截图页：关键 UI 是否可读，PiP 是否遮挡重点。
- 检查底部黄色字幕是否裁切、是否超过 20 字、是否和当前口播一致。
- 检查大标题是否有足够冲击力，关键词是否被颜色和动效强调。
- 检查装饰元素是否只是辅助，不得压住主标题、字幕和 PiP。
