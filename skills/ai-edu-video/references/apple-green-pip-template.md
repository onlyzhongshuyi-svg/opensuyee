# Apple Green PiP 模板

这个模板用于竖屏 AI 科普/教程类视频。整体感觉要简洁、清晰、有科技感，像 Apple 官网教程页面被改造成短视频：留白充足、主次明确、口播画面在右下角稳定露出。

## 视觉规格

- 视频比例：3:4 竖屏，推荐导出 `1080x1440`。
- 风格：干净、轻盈、白色/浅灰绿色背景，细网格，轻微绿色光晕，少量科技感点缀。
- 主色：优先取用户提供的品牌色或色卡；如果没有，使用与产品调性匹配的主色。
- 点缀色：可以少量使用青色、黄色等辅助色，但不要抢主色。
- 字体：标题大而清楚，正文紧凑，不要出现拥挤段落。
- 如果顶部有章节进度条，不要再放拥挤的右上角水印。

## 封面开场

- 前 1-3 秒可以做成独立封面帧，不必沿用正文页布局。
- 标题要明显大于正文页标题，适合暂停后当视频封面看。
- 根据本期产品生成标题和副标题，例如：
  - 标题：`产品名 入门教程`
  - 副标题：面向小白的结果承诺，例如 `小白也能用 AI 做项目赚钱`
- 标题块建议占画面高度约 25%-35%。
- 产品 icon/logo 规则：
  - 用户提供产品 icon/logo 时，放入封面标题区或标题旁边。
  - 用户没有提供 icon/logo 时，不要强行放固定产品图标；可以用产品名文字标识、抽象符号或留空。
  - 当前讲 Codex 才使用 Codex icon；讲其他产品时换成对应产品 icon。
- 如果用户要求口播视频在封面露出，右下角 PiP 从第 0 秒开始显示。
- 右下角要留出安全区域，避免标签、字幕、卡片被 PiP 遮挡。

## 正文布局

- 顶部吸顶章节进度条：
  - 进度条长度对应完整视频时长。
  - 如果口播有 3 个部分，分成 3 段。
  - 文案来自本期脚本的真实章节，例如：
    - `产品是什么`
    - `如何安装`
    - `从0开始做第1个项目`
  - 进度条和主标题之间要留足距离。
- 右下角口播 PiP：
  - 位置全片稳定。
  - 可以加细绿色描边。
  - 不要把字幕或关键信息放到 PiP 下方。
- 主内容区：
  - 标题使用大字，并用主色高亮重点词。
  - 卡片、流程、表格、终端、产品截图等模块跟随口播逐步出现。
  - 画面内容要和当前口播句子一一对应。

## 场景模式

- **开场承诺**：大标题 + 绿色重点词 + 简短结果承诺。
- **本期结构**：3 个模块或编号步骤。
- **概念对比**：两行或多行对比表，例如 `ChatGPT / chat`、`Codex / agent`。
- **能做什么**：官网、PPT、APP、自动化、产品案例等卡片。
- **安装步骤**：账号、官网、下载、安装、登录、权限设置。
- **权限说明**：用小面板、开关、分段控件表达。
- **第一个项目**：提示词气泡、终端/代码风格块、结果预览。
- **结尾总结**：恭喜入门 + 下一步行动。

## 口播与画面对齐

- 先建立句子级时间轴，再设计页面。
- 按句子或短语拆分口播稿，切分优先级：
  - 明显停顿。
  - 句号、逗号、顿号。
  - “首先、然后、下面、比如、最后”等步骤词。
  - 产品名、核心概念、数字承诺和结果承诺。
- 每个短句至少带来一个画面变化：出现一张卡、强调一个关键词、推进一个步骤、更新一个模块。
- 句子级时间轴建议使用这个结构：
  - `start`
  - `end`
  - `spoken_text`
  - `caption_text`
  - `visual_action`
  - `scene_id`
- 页面/场景级时间轴建议使用这个结构：
  - `scene_start`
  - `scene_end`
  - `title`
  - `visual_type`
  - `stage_count`
  - `sentences`
- 如果用户反馈“图和音频对不上”，要围绕该时间点抽帧，并调整场景时长或场景内容。
- 对用户明确指出的时间点，必须在导出后抽帧检查。

## 字幕

- 每一行中文字幕不超过 20 个字。
- 使用黑色字幕底框、白色文字。
- 黑色底框宽度跟随字幕实际长度，不要短字幕配长底框。
- 优先 1 行字幕；必要时最多 2 行。

## 本模板从 Codex 教程沉淀出的注意点

- 顶部进度条拥挤时，去掉右上角产品水印。
- 概念对比页不要出现用户已要求删除的旧文案。
- 产品对比中的英文标签要按用户要求精确修改，例如 `work` 改为 `agent`。
- 封面是否放 icon 取决于本期产品和用户素材，不固定为 Codex icon。
- 用户要求封面露出口播视频时，PiP 从第 0 秒开始显示。

## 验收清单

- 用 `ffprobe` 检查 `1080x1440`、`30fps`、预期时长和音频流。
- 抽帧检查：
  - 1 秒：封面标题、副标题、产品 icon/logo（如果提供）、口播 PiP。
  - 2-3 秒：进入正文布局后的进度条、标题、字幕、PiP。
  - 用户指定的时间点，例如 34 秒。
  - 章节切换点。
- 确认字幕短、不重叠、黑底框自适应宽度。
- 确认顶部进度条文案完整、不拥挤。
- 确认 PiP 没有遮挡关键信息。
